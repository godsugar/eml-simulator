# EML演算子の概念

## 定義

**EML**（Exp-Minus-Log）演算子は、次のように定義される二項演算子:

```
eml(x, y) = exp(x) − ln(y)
```

指数関数と自然対数を組み合わせた1つの演算子であり、これだけで全ての初等関数を表現できる。

---

## 文法

EML式は以下の最小文法で生成される:

```
S → 1 | x | eml(S, S)
```

- 葉（リーフ）は定数 `1` と変数 `x` のみ
- 内部ノードはすべて同じ `eml` 演算子
- 任意の深さの二分木を構成できる

---

## なぜ1演算子で十分か

### exp と ln の相殺

`eml` の右辺に `1` を置くと `ln(1) = 0` で消え、指数関数だけが残る:

```
eml(x, 1) = exp(x) − ln(1) = exp(x)
```

逆に、入れ子にすることで `exp` と `ln` が相殺し合い、対数関数が取り出せる。

### 複素数の活用

複素数の指数関数にはオイラーの公式が成り立つ:

```
exp(ix) = cos(x) + i·sin(x)
```

これにより三角関数・逆三角関数も EML の入れ子で表現できる。

### 入れ子の深さの目安（本シミュレータ実測値）

| 関数 | リーフ数 K | depth |
|------|-----------|-------|
| exp(x), e | 2 | 1 |
| ln(x), 0 | 4 | 3 |
| x, x−1 | 5〜6 | 4 |
| 1/x, e⁻¹, exp(−x), 2 | 9〜14 | 8 |
| x², x³, x⁴ | 13〜37 | 6〜8 |
| i | 34 | 19 |
| π | 53 | 25 |
| sin(x), cos(x) | 135〜172 | 36〜38 |
| tan(x) | 314 | 44 |

## 発見の経緯

論文著者 Andrzej Odrzywołek によると、このアイデアは**系統的な全探索**によって発見されたものであり、事前に予想されていたわけではない。一様な二分木構造により、勾配法を用いた記号回帰（symbolic regression）にも応用できる。

## 総当たり探索による深さ検証

定数 1 のみをリーフに持つ式の値集合を深さごとに構築した:

```
V(0) = {1}
V(d) = V(d-1) ∪ { exp(a) − ln(b) | a, b ∈ V(d-1) }
```

| depth | ユニーク値数 |
|-------|------------|
| 0 | 1 |
| 1 | 3 |
| 2 | 16 |
| 3 | 107 |
| 4 | 757 |
| 5 | 36,987 |

depth 5 までの全値を列挙した結果、π や i に近い値は存在するが数値精度の範囲では一致しなかった。
depth 6 は meet-in-the-middle でも厳密一致は見つからなかった。本シミュレータの実装では i は depth 19、π は depth 25 で構成している。
