# EML演算子シミュレータ 要件定義

## 概要

論文 **"All Elementary Functions from a Single Operator"** (arXiv:2603.21852, Andrzej Odrzywołek) で定義された EML演算子のインタラクティブシミュレータ。

ブラウザの静的ファイルだけで動作する Next.js SSG アプリケーション。

---

## EML演算子の定義

```
eml(x, y) = exp(x) − ln(y)
```

### 文法

```
S → 1 | x | eml(S, S)
```

定数 `1` と変数 `x` のみを葉（リーフ）として、EMLの入れ子で全ての初等関数を表現できる。

### 構成例（論文 Table 4 より）

| 関数 | EML式 | リーフ数 K |
|------|-------|-----------|
| exp(x) | `eml(x, 1)` | 2 |
| e | `eml(1, 1)` | 2 |
| ln(x) | `eml(1, eml(eml(1, x), 1))` | 4 |
| 0 | `eml(1, eml(eml(1, 1), 1))` | 4 |

## 機能要件

### FR-1: 式木ビルダー
- 空スロット `_` からスタートし、クリックで `1` / `x` / `eml(_, _)` を選択
- 入れ子は任意の深さまで対応
- ノード削除（スロットに戻す）が可能
- 式全体の括弧表記をリアルタイム表示
- 論文のリーフ数 K を表示

### FR-2: プリセット式
- 論文記載の主要な式をワンクリックでロード
- 対応: `exp(x)`, `e`, `ln(x)` など

### FR-3: 変数 x の入力
- 実部・虚部の数値入力（複素数対応）
- クイック選択: `1`, `-1`, `i`, `-i`, `0`, `e`, `π`
- 実軸上のスライダーモード

### FR-4: 評価・ステップ表示
- 式木の各ノードの中間評価結果を表示
- `ln(0)` は未定義として適切に表示

### FR-5: 複素平面プロット
- 最終評価結果を複素平面上にプロット
- x をスライダーで変化させると軌跡を描画

---

## 非機能要件

### NFR-1: 静的ファイルのみ
- `next build` → `out/` ディレクトリのみで動作
- サーバーサイド処理なし

### NFR-2: UI言語
- 日本語

### NFR-3: 複素数演算
- 外部ライブラリ不使用、自前実装
- `exp(a+bi) = e^a(cos b + i sin b)`
- `ln(a+bi) = ln|z| + i·arg(z)`
